ES6 之后写 JS 基本每行都在用这些语法，但偶尔还是会在某个细节上卡一下。这里记的都是我觉得值得记的部分，不是所有语法的完整说明。

## let / const vs var

```javascript
// var 有变量提升，函数作用域，容易出问题
console.log(a) // undefined（不报错）
var a = 1

// let/const 块级作用域，没有提升，推荐用
console.log(b) // ❌ ReferenceError
let b = 1
```

**原则：** 默认用 `const`，需要重新赋值再用 `let`，不用 `var`。

`const` 只是不能重新赋值，对象/数组的内容还是可以改的：

```javascript
const obj = { count: 0 }
obj.count++ // ✅ 可以
obj = {}    // ❌ 报错
```

## 解构赋值

### 数组解构

```javascript
const [a, b, c] = [1, 2, 3]
// a=1, b=2, c=3

// 跳过某项
const [, second] = [1, 2, 3]
// second=2

// 默认值
const [x = 0, y = 0] = [10]
// x=10, y=0

// 交换变量
let m = 1, n = 2
;[m, n] = [n, m]
// m=2, n=1
```

### 对象解构

```javascript
const user = { name: '张三', age: 25, role: 'admin' }

const { name, age } = user
// name='张三', age=25

// 重命名
const { name: userName } = user
// userName='张三'

// 默认值
const { avatar = '/default.png' } = user
// avatar='/default.png'

// 函数参数解构
function greet({ name, age = 0 }) {
  return `${name}, ${age}岁`
}
greet(user) // '张三, 25岁'
```

## 模板字符串

```javascript
const name = '张三'
const score = 98.5

// ❌ 旧写法拼接麻烦
'你好，' + name + '，你的得分是 ' + score + ' 分'

// ✅ 模板字符串
`你好，${name}，你的得分是 ${score} 分`

// 可以放表达式
`总分：${score > 90 ? '优秀' : '良好'}`

// 多行字符串
const html = `
  <div>
    <p>${name}</p>
  </div>
`
```

## 箭头函数

```javascript
// 普通函数
function add(a, b) { return a + b }

// 箭头函数
const add = (a, b) => a + b

// 只有一个参数可以省括号
const double = n => n * 2

// 函数体多行加花括号，要显式 return
const process = (n) => {
  const result = n * 2
  return result + 1
}

// 返回对象要加括号（不然花括号会被当函数体）
const toObj = (x) => ({ value: x })
```

箭头函数没有自己的 `this`，继承外层作用域的 `this`，适合回调函数：

```javascript
// ❌ 普通函数 this 丢失
class Timer {
  start() {
    setTimeout(function() {
      this.tick() // this 不是 Timer 实例
    }, 1000)
  }
}

// ✅ 箭头函数继承 this
class Timer {
  start() {
    setTimeout(() => {
      this.tick() // this 是 Timer 实例
    }, 1000)
  }
}
```

## 扩展运算符 / 剩余参数

```javascript
// 展开数组
const a = [1, 2, 3]
const b = [0, ...a, 4]
// [0, 1, 2, 3, 4]

// 合并对象（后面的会覆盖前面的同名属性）
const defaults = { theme: 'dark', lang: 'zh' }
const custom = { lang: 'en', size: 14 }
const config = { ...defaults, ...custom }
// { theme: 'dark', lang: 'en', size: 14 }

// 浅拷贝
const copy = { ...original }
const arrCopy = [...arr]

// 剩余参数（放最后）
function sum(first, ...rest) {
  return rest.reduce((acc, n) => acc + n, first)
}
sum(1, 2, 3, 4) // 10
```

## 默认参数

```javascript
// ❌ 旧写法
function request(url, method) {
  method = method || 'GET'
}

// ✅ 默认参数
function request(url, method = 'GET', timeout = 5000) {
  // ...
}

request('/api/user')               // GET, 5000
request('/api/user', 'POST')       // POST, 5000
request('/api/user', 'POST', 3000) // POST, 3000
```

传 `undefined` 会用默认值，传 `null` 不会：

```javascript
request('/api', undefined, 3000) // method='GET'（触发默认值）
request('/api', null, 3000)      // method=null（不触发）
```

## 可选链 ?. 和空值合并 ??

ES2020，但现在用得非常普遍，不能不提。

```javascript
const user = { profile: { avatar: 'url' } }

// ❌ 层层判断很烦
user && user.profile && user.profile.avatar

// ✅ 可选链
user?.profile?.avatar // 'url'
user?.address?.city   // undefined（不报错）

// 方法调用
user?.getPermissions?.()

// 数组
arr?.[0]
```

```javascript
// ?? 只在 null/undefined 时用默认值（和 || 不同，|| 在 0/'' 时也会用默认值）
const count = data.count ?? 0    // data.count 是 0 时保留 0
const count2 = data.count || 0   // data.count 是 0 时变成 0（❌ 可能不符合预期）

const name = user.name ?? '匿名'
```

## Promise

解决回调地狱，链式调用异步操作。

```javascript
// 创建
const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve('done'), 1000)
})

// 使用
p.then(result => console.log(result))
 .catch(err => console.error(err))
 .finally(() => console.log('无论如何都执行'))

// async/await：更直观的写法
async function fetchUser(id) {
  try {
    const res = await fetch(`/api/user/${id}`)
    const data = await res.json()
    return data
  } catch (err) {
    console.error(err)
  }
}
```

### 常用的 Promise 静态方法

| 方法 | 触发时机 | 失败行为 | 典型场景 |
|------|----------|----------|----------|
| `Promise.all` | 全部成功 | 任一失败则整体 reject | 并行请求、互相依赖的数据 |
| `Promise.race` | 第一个完成（无论成败） | 第一个失败则 reject | 超时控制、竞速取最快 |
| `Promise.allSettled` | 全部完成 | 不短路，收集所有结果 | 批量操作、统计成功/失败数 |
| `Promise.any` | 第一个成功 | 全部失败才 reject | 备用服务、多源 CDN |

**`Promise.all` — 全部成功才继续，任一失败则中止**

```javascript
// 用户页需要同时加载用户信息和订单，缺一不可
const [user, orders] = await Promise.all([
  fetchUser(id),
  fetchOrders(id),
])
```

适合：请求之间**互相依赖**，或结果必须同时存在才有意义。

**`Promise.race` — 第一个完成（成功或失败）就结束**

```javascript
// 5 秒内没响应就超时报错
const result = await Promise.race([
  fetchData(),
  new Promise((_, reject) => setTimeout(() => reject(new Error('timeout')), 5000)),
])
```

适合：**超时控制**。注意：输掉的 Promise 仍在后台运行，不会被取消。

**`Promise.allSettled` — 等全部完成，不管成败**

```javascript
// 批量上传图片，统计结果
const results = await Promise.allSettled([upload(img1), upload(img2), upload(img3)])
const succeeded = results.filter(r => r.status === 'fulfilled').length
const failed    = results.filter(r => r.status === 'rejected')
console.log(`成功 ${succeeded}，失败 ${failed.length}`)
```

适合：**批量操作**，需要知道每一项的结果，不能因为某一个失败就中断其他。

**`Promise.any` — 第一个成功就结束，全败才报错**

```javascript
// 从多个 CDN 取资源，哪个快用哪个
const img = await Promise.any([
  fetch('https://cdn1.example.com/img.png'),
  fetch('https://cdn2.example.com/img.png'),
])
```

适合：**备用方案**，多个来源竞争，只要有一个成功即可。

**实战：定投金额校验**

校验用户输入的定投金额，需要同时满足：① ≥ 起购金额；② ≤ 可用余额。
其中起购金额来自产品详情接口，而产品详情接口需要先用 `productCode` 查出 `productId` 才能调用——形成依赖链。

```javascript
async function validateAmount(productCode, inputAmount) {
  // 第一步：拿到 productId（后续查询的前置依赖）
  const { productId } = await fetchProductInfo(productCode)

  // 第二步：productId 到手后，起购金额和余额可以并行查
  const [minAmount, balance] = await Promise.all([
    fetchMinInvestAmount(productId),   // 依赖 productId，不能提前
    fetchAvailableBalance(),            // 与产品无关，可以并行
  ])

  if (inputAmount < minAmount) {
    return { valid: false, reason: `最低起购金额为 ${minAmount} 元` }
  }
  if (inputAmount > balance) {
    return { valid: false, reason: `可用余额不足，当前余额 ${balance} 元` }
  }
  return { valid: true }
}
```

关键点：`fetchProductInfo` 必须先完成（链式 `await`），拿到 `productId` 之后，两个校验数据之间没有依赖关系，用 `Promise.all` 并行发出，总耗时取决于较慢的那个，而不是两者之和。

## Map / Set

### Set — 不重复的集合

```javascript
// 数组去重
const arr = [1, 2, 2, 3, 3]
const unique = [...new Set(arr)]
// [1, 2, 3]

const set = new Set([1, 2, 3])
set.add(4)
set.has(2) // true
set.delete(2)
set.size   // 3
```

### Map — 键可以是任意类型的对象

```javascript
const map = new Map()
map.set('name', '张三')
map.set(1, 'one')
map.set({ key: true }, 'obj-key')

map.get('name') // '张三'
map.has('name') // true
map.size        // 3

// 遍历
map.forEach((value, key) => console.log(key, value))
for (const [key, value] of map) { ... }
```

比普通对象好的地方：键可以是对象，有 `size`，遍历顺序固定（按插入顺序）。
