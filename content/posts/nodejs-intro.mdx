Node.js 让 JS 能在浏览器外跑。装好后 `node hello.js` 就能直接执行，不需要编译，不需要打包。

## 我实际用它干了什么

### 1. 写构建脚本

脚本有两种用法：

- **集成进项目**：放 `scripts/deploy.mjs`，`package.json` 里注册 `"deploy": "node scripts/deploy.mjs"`，`npm run deploy` 触发，属于项目代码，进 git
- **临时跑一次**：随便建个 `convert.js`，`node convert.js` 跑完就删，不用注册

### 2. 处理文件

做这个博客需要把地球贴图批量转格式，写了个临时脚本：

```js
const { execSync } = require('child_process')
const fs = require('fs')

fs.readdirSync('./public').forEach(file => {
  if (file.endsWith('.png')) {
    execSync(`sips -s format webp ./public/${file} --out ./public/${file.replace('.png', '.webp')}`)
  }
})
```

跑完就删，不进 git。

### 3. 跑本地服务

临时起一个静态文件服务：

```js
const http = require('http')
const fs = require('fs')
const path = require('path')

http.createServer((req, res) => {
  const file = path.join('./dist', req.url === '/' ? 'index.html' : req.url)
  fs.existsSync(file) ? res.end(fs.readFileSync(file)) : (res.writeHead(404), res.end('not found'))
}).listen(8080)

console.log('http://localhost:8080')
```

### 4. 接口 mock

联调等接口时本地起个 mock：

```js
const http = require('http')

const data = {
  '/api/user': { id: 1, name: '王一贺', role: 'admin' },
  '/api/posts': [{ id: 1, title: '测试文章' }],
}

http.createServer((req, res) => {
  res.setHeader('Content-Type', 'application/json')
  res.setHeader('Access-Control-Allow-Origin', '*')
  res.end(JSON.stringify(data[req.url] ?? { error: 'not found' }))
}).listen(3001)
```

如果团队共用就放进 git，个人临时用直接跑完删。后来换成 `json-server` 了，原理一样。

## 踩过的坑

**`require` vs `import`**：Node.js 默认 CommonJS，文件名 `.mjs` 或 `package.json` 设 `"type": "module"` 才能用 `import`。现在用 TypeScript 就没这烦恼了。

**异步**：文件、网络操作都是异步的，早期没搞清楚，取到的值是 `undefined`。后来习惯了 `async/await`，小脚本直接用 `fs.readFileSync`。

**路径**：`./file.json` 相对的是**执行命令时的目录**，不是脚本文件位置。用 `path.join(__dirname, '../data.json')` 才稳。

## 常用的几个

| 场景 | 用法 |
|------|------|
| 读写文件 | `fs.readFileSync` / `fs.writeFileSync` |
| 路径拼接 | `path.join(__dirname, '../...')` |
| 执行命令 | `child_process.execSync('git status')` |
| 环境变量 | `process.env.NODE_ENV` |
| 项目根目录 | `process.cwd()` |
