## 支付流程梳理

H5 里做微信支付主要有两种：

- **JSAPI 支付**：在微信内置浏览器里，调 `wx.chooseWXPay()` 或 `WeixinJSBridge.invoke()`，体验最好
- **H5 支付**：在外部浏览器里，后端返回 `mweb_url`，前端直接跳过去，微信处理完自动跳回

这篇说的是 JSAPI，坑比较多。

---

## 流程图

```
用户点击支付
  ↓
前端调后端接口获取 prepay_id 和签名参数
  ↓
前端调 WeixinJSBridge.invoke('getBrandWCPayRequest', params, callback)
  ↓
微信唤起收银台
  ↓
用户确认支付
  ↓
回调函数里拿到 res.err_msg
  ↓
前端轮询后端确认实际支付状态
```

最后一步**轮询**很关键，不能只靠前端的 `err_msg === 'get_brand_wcpay_request:ok'` 来判断支付成功——微信支付的回调不是 100% 可靠。

## 前端代码

```typescript
function wxPay(params: {
  appId: string
  timeStamp: string
  nonceStr: string
  package: string      // prepay_id=xxx
  signType: 'RSA' | 'MD5'
  paySign: string
}): Promise<'ok' | 'cancel' | 'fail'> {
  return new Promise((resolve, reject) => {
    if (!window.WeixinJSBridge) {
      reject(new Error('请在微信内打开'))
      return
    }

    WeixinJSBridge.invoke('getBrandWCPayRequest', params, (res) => {
      if (res.err_msg === 'get_brand_wcpay_request:ok') {
        resolve('ok')
      } else if (res.err_msg === 'get_brand_wcpay_request:cancel') {
        resolve('cancel')
      } else {
        resolve('fail')
      }
    })
  })
}
```

## 我碰到的那几个坑

### 坑一：WeixinJSBridge 不存在

微信内置浏览器打开页面时，`WeixinJSBridge` 不一定立刻就绪，有时候页面渲染完了它还没注入。

正确做法是监听 `WeixinJSBridgeReady` 事件：

```typescript
function waitForBridge(): Promise<void> {
  if (window.WeixinJSBridge) return Promise.resolve()

  return new Promise(resolve => {
    document.addEventListener('WeixinJSBridgeReady', () => resolve(), { once: true })
  })
}

// 使用
await waitForBridge()
await wxPay(params)
```

### 坑二：签名参数里的 package 字段名

JSAPI 参数里有个字段叫 `package`，值是 `prepay_id=xxx`。这个字段名是 JS 保留关键字，有些写法直接用对象字面量会报语法错误：

```javascript
// ❌ 在某些旧环境下可能有问题
const params = {
  package: 'prepay_id=xxx',  // package 是保留字
}

// ✅ 安全写法
const params: Record<string, string> = {}
params['package'] = 'prepay_id=xxx'
```

### 坑三：timeStamp 必须是字符串

微信 SDK 的 `timeStamp` 参数类型是字符串，不是数字。后端如果返回的是 `number`，前端记得转：

```typescript
timeStamp: String(data.timeStamp),
```

传数字进去会导致签名验证失败，报 `chooseWXPay:fail` 但错误信息不明显，很难排查。

### 坑四：支付回调不可信

`res.err_msg === 'get_brand_wcpay_request:ok'` 只代表微信 SDK 告诉你"用户操作完了"，不代表钱真的到了。网络异常、掉单等情况都可能导致前端拿到 ok 但实际支付失败。

正确做法是支付后轮询后端订单状态：

```typescript
async function pollOrderStatus(orderId: string, maxTries = 5): Promise<'paid' | 'unpaid'> {
  for (let i = 0; i < maxTries; i++) {
    const { status } = await getOrderStatus(orderId)
    if (status === 'paid') return 'paid'
    await sleep(1000 * (i + 1))  // 间隔递增
  }
  return 'unpaid'
}
```

## 环境判断

有时候需要区分是微信内还是外部浏览器：

```typescript
export const isWeixin = /MicroMessenger/i.test(navigator.userAgent)
export const isWeixinMiniProgram = isWeixin && /miniprogram/i.test(navigator.userAgent)
```

小程序里的 webview 和微信浏览器支付方式不一样，小程序要用 `wx.requestPayment`，不能用 JSAPI。
