写了很久的 `forEach`，有一次自己重构一段数据处理代码，想着能不能少写几行，翻了翻这几个方法，才发现之前一直在用最低效的方式。`forEach` 只是个迭代器，`map`/`filter`/`reduce` 才是真正的数据变换工具。

## map / filter / reduce 的本质

这三个方法都是对数组的变换，区别在于输出：

- `map`：一对一变换，输入 n 个元素，输出 n 个
- `filter`：筛选，输出是输入的子集
- `reduce`：折叠，把数组归约成任意类型的值（不一定是数组）

`reduce` 是最通用的，`map` 和 `filter` 都可以用它实现：

```js
const arr = [1, 2, 3, 4, 5]

// 用 reduce 实现 map
const doubled = arr.reduce((acc, x) => {
  acc.push(x * 2)
  return acc
}, [])
// doubled → [2, 4, 6, 8, 10]

// 用 reduce 实现 filter
const evens = arr.reduce((acc, x) => {
  if (x % 2 === 0) acc.push(x)
  return acc
}, [])
// evens → [2, 4]
```

理解这一点之后，看到 `reduce` 就不慌了，知道它本质上就是「带着一个累积值遍历数组」。

## map：批量格式化展示字段

H5 项目里有一个很典型的场景：后端返回的是原始数据（timestamp、raw amount 字符串），前端展示时要批量格式化。每条记录都要处理，用 `map` 最合适。

项目里用的是 `WhaleSDK.Fmt` 这套格式化工具，`Fmt.date` 处理时间，`Fmt.financialNumber` 处理金额：

```js
// 后端原始数据
const records = [
  { id: 1, created_at: 1703001600, amount: '1234.56', status: 'completed' },
  { id: 2, created_at: 1703088000, amount: '567.89',  status: 'pending'   },
]

// 用 map 一次性转换所有展示字段
const formatted = records.map(record => ({
  ...record,
  dateStr:   WhaleSDK.Fmt.date(record.created_at * 1000, 'date_time_simple'),
  amountStr: `$${WhaleSDK.Fmt.financialNumber(record.amount)}`,
}))

// formatted[0] →
// {
//   id: 1,
//   created_at: 1703001600,
//   amount: '1234.56',
//   status: 'completed',
//   dateStr: '2023/12/20 08:00',      ← 时间格式化
//   amountStr: '$1,234.56',           ← 千分位金额格式化
// }
```

相比 `forEach` 逐条 push，`map` 的意图更清晰：「我要把这个数组的每条数据都转成新形状」。

### Fmt.date 的实现思路

`Fmt.date` 的第一个参数类型是 `dayjs.ConfigType`，这是关键。dayjs 的 ConfigType 本身就支持多种输入：

```js
// 这几种写法都能正确解析
WhaleSDK.Fmt.date(1703001600000)          // 毫秒时间戳
WhaleSDK.Fmt.date(1703001600 * 1000)      // 秒级时间戳转毫秒（后端常见）
WhaleSDK.Fmt.date(new Date())             // Date 对象
WhaleSDK.Fmt.date('2023-12-20')           // 日期字符串
WhaleSDK.Fmt.date('2023-12-20T08:00:00Z') // ISO 字符串

// 第二个参数是预置格式名，而不是自己写 'YYYY/MM/DD HH:mm'
WhaleSDK.Fmt.date(ts, 'date_simple')       // → '2023/12/20'
WhaleSDK.Fmt.date(ts, 'date_time_simple')  // → '2023/12/20 08:00'
WhaleSDK.Fmt.date(ts, 'date_short')        // → '2023年12月20日'（中文）/ 'Dec 20, 2023'（英文）
```

这样设计有两个好处：

**第一，自动多语言。** 同一个 `'date_short'`，在中文环境输出 `2023年12月20日`，在英文环境输出 `Dec 20, 2023`，调用方不需要关心当前语言，也不需要为每种语言写不同的格式字符串。

**第二，语义化格式名。** 自己写 `'YYYY/MM/DD HH:mm'` 需要记格式符，也不知道这个格式在不同语言下长什么样。用 `'date_time_simple'` 一看就知道是「简洁的日期+时间」，维护时更直观。

在 `map` 里批量格式化的时候，好处更明显——每条记录都走同一套逻辑，格式统一、语言统一，不会出现某些记录格式不一样的情况。

手机号脱敏也是类似的：

```js
const phones = ['13800138000', '13912345678', '15866667777']

const masked = phones.map(p => WhaleSDK.Fmt.phone(p, { desensitize: true }))
// ['138****8000', '139****5678', '158****7777']
```

## filter + reduce：筛选后聚合

`filter` 和 `reduce` 组合起来非常强。先筛选出感兴趣的数据，再做聚合计算：

```js
// 统计 completed 状态的总金额
const completedTotal = records
  .filter(r => r.status === 'completed')
  .reduce((sum, r) => sum + Number(r.amount), 0)
// completedTotal → 1234.56

// 只展示有效的转账记录（排除失败的）
const validRecords = records.filter(r => r.status !== 'failed')
```

## reduce：先搞懂参数

`reduce` 的完整签名：

```js
array.reduce(callback(accumulator, currentValue, currentIndex, array), initialValue)
```

四个参数，常用的是前两个：

| 参数 | 含义 |
|------|------|
| `accumulator` | 累积值，上一次回调返回的结果（第一次是 `initialValue`） |
| `currentValue` | 当前元素 |
| `currentIndex` | 当前元素的索引（可选） |
| `array` | 原数组本身（可选，很少用） |
| `initialValue` | 初始值，决定 `accumulator` 第一次是什么 |

**initialValue 非常关键**，它决定了 `acc` 的初始类型，也决定了最终的输出类型：

```js
const nums = [1, 2, 3, 4]

// initialValue 是 0 → 累加求和，最终是 number
nums.reduce((acc, n) => acc + n, 0)   // 10

// initialValue 是 [] → 累积数组，最终是 array
nums.reduce((acc, n) => {
  acc.push(n * 2)
  return acc
}, [])  // [2, 4, 6, 8]

// initialValue 是 {} → 累积对象，最终是 object
nums.reduce((acc, n) => {
  acc[`key_${n}`] = n * n
  return acc
}, {})  // { key_1: 1, key_2: 4, key_3: 9, key_4: 16 }
```

每次回调**必须 return acc**，否则下一次的 `acc` 会是 `undefined`，这是最常见的错误：

```js
// ❌ 忘记 return，acc 第二次开始就是 undefined
nums.reduce((acc, n) => {
  acc.push(n)  // 没有 return
}, [])

// ✅ 一定要 return
nums.reduce((acc, n) => {
  acc.push(n)
  return acc  // ← 不能省
}, [])
```

不传 `initialValue` 时，第一个元素自动作为初始值，循环从第二个元素开始，只适合类型统一的简单累加，其他情况都建议明确传：

```js
// 可以不传，但只适合这种简单场景
[1, 2, 3].reduce((acc, n) => acc + n)  // 6，第一次 acc=1，n=2
```

## reduce：把数组转成对象（O(1) 查找）

这是我用 `reduce` 最多的场景。

O(1) 是复杂度表示，意思是不管数组有多长，都能直接通过 key 定位到数据，不需要遍历。相比之下，`list.find(item => item.id === 'xxx')` 每次都要从头扫，数组越长越慢（O(n)）。

```js
const list = [
  { id: 'fund_001', name: '沪深300ETF', nav: 1.234 },
  { id: 'fund_002', name: '纳斯达克ETF', nav: 2.567 },
  { id: 'fund_003', name: '恒生科技ETF', nav: 0.891 },
]

// 转成以 id 为 key 的对象
const fundMap = list.reduce((acc, item) => {
  acc[item.id] = item
  return acc
}, {})

// fundMap →
// {
//   fund_001: { id: 'fund_001', name: '沪深300ETF', nav: 1.234 },
//   fund_002: { id: 'fund_002', name: '纳斯达克ETF', nav: 2.567 },
//   fund_003: { id: 'fund_003', name: '恒生科技ETF', nav: 0.891 },
// }

// 之后按 id 查找，直接取值，不用 find
console.log(fundMap['fund_002'].name)  // '纳斯达克ETF'
console.log(fundMap['fund_003'].nav)   // 0.891
```

按条件分组也是一样的思路：

```js
const orders = [
  { id: 1, status: 'completed', amount: 100 },
  { id: 2, status: 'pending',   amount: 200 },
  { id: 3, status: 'completed', amount: 150 },
  { id: 4, status: 'failed',    amount: 80  },
]

const grouped = orders.reduce((acc, order) => {
  const key = order.status
  if (!acc[key]) acc[key] = []
  acc[key].push(order)
  return acc
}, {})

// grouped →
// {
//   completed: [{ id: 1, ... }, { id: 3, ... }],
//   pending:   [{ id: 2, ... }],
//   failed:    [{ id: 4, ... }],
// }
```

## 柯里化：参数预置

柯里化是把多参数函数变成一系列单参数函数。我用它最多的地方是事件处理函数的参数预置：

```js
// 不用柯里化：每次都要写箭头函数包一层
items.map(item => (
  <button onClick={() => handleDelete(item.id)}>删除</button>
))

// 柯里化版本：handleDelete(id) 返回一个函数，直接作为 onClick
const handleDelete = (id: string) => (e: React.MouseEvent) => {
  e.stopPropagation()
  deleteItem(id)
}

items.map(item => (
  <button onClick={handleDelete(item.id)}>删除</button>
))
```

`handleDelete(item.id)` 预置了 id，返回一个事件处理函数，不需要在 JSX 里再套一层箭头函数。

## 链式调用的可读性

`map` / `filter` / `reduce` 可以链式组合，但不要为了链式而链式：

```js
// 清晰：筛选活跃用户，取出名字
const activeNames = users
  .filter(u => u.active)
  .map(u => u.name)

// 这种直接写就够了，不需要包装成什么
```

如果链太长、中间步骤需要复用，提取成命名函数比链式调用更容易维护。
