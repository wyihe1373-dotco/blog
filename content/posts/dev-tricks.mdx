开发效率很大程度上取决于工具熟不熟。这里记录一下我日常用到最多的技巧，不是大而全的清单，是真正高频用到的那些。

## Chrome DevTools

### console 不只有 log

```js
// 带标签输出对象，展开后能看到字段
console.log('user:', user)

// 表格格式，数组对象一目了然
console.table(fundList)
// 输出：
// ┌─────────┬────────────┬──────────┬───────┐
// │ (index) │     id     │   name   │  nav  │
// ├─────────┼────────────┼──────────┼───────┤
// │    0    │ 'fund_001' │ '沪深300' │ 1.234 │
// │    1    │ 'fund_002' │ '纳指ETF' │ 2.567 │
// └─────────┴────────────┴──────────┴───────┘

// 分组折叠
console.group('请求详情')
console.log('url:', url)
console.log('params:', params)
console.groupEnd()

// 计时
console.time('接口耗时')
await fetchData()
console.timeEnd('接口耗时')
// 接口耗时: 234.12ms
```

### 断点调试代替 console.log

`console.log` 只能打印当时的值，遇到异步或者循环逻辑容易看不清楚。断点能暂停执行、查看完整调用栈。

在代码里加 `debugger` 语句，打开 DevTools 后运行到那里会自动暂停：

```js
async function loadFundData(code) {
  const raw = await api.getFund(code)
  debugger  // 在这里暂停，左侧 Scope 面板能看到 raw 的完整内容
  const formatted = formatFund(raw)
  return formatted
}
```

也可以直接在 Sources 面板点行号打断点，不用改代码。**条件断点**更实用：右键行号 → "Add conditional breakpoint"，填 `code === 'fund_001'`，只在满足条件时才停，不用每次都手动跳过。

### Network 面板

**模拟慢网络**：Network 面板顶部 → Throttle → 选 "Slow 3G"，能复现用户弱网下的体验，骨架屏和 loading 态测试必用。

**Copy as fetch**：右键某个请求 → "Copy" → "Copy as fetch"，粘贴到 console 直接重放这个请求，调接口时不用重新触发页面操作。

**XHR/Fetch 过滤**：Filter 里输入 `-` 可以排除，比如 `-favicon` 把 favicon 请求过滤掉，只看业务接口。

### Elements 面板

选中元素后直接在 Styles 面板改 CSS，改完复制出来。比在代码里改→保存→刷新的循环快很多。

**强制 hover/focus 状态**：选中元素 → 右上角 `:hov` → 勾选 `:hover`，不需要鼠标悬停就能看 hover 样式，调下拉菜单特别好用。

## VS Code

### 多光标编辑

`Option + Click`（Mac）在多处同时放光标，然后一起编辑。改多个变量名、批量加前缀都能用。

`Cmd + D` 选中当前词，再按一次选下一个相同的词，依次添加光标，批量重命名很快。

### 快速定位

`Cmd + P`：按文件名跳转，输入文件名关键词，不用在侧边栏一层层找。

`Cmd + Shift + O`：在当前文件里按符号（函数名、变量名）跳转，大文件里定位函数很快。

`Cmd + Click`：跳转到定义，比手动搜索快。

### 终端操作

`Ctrl + 反引号`：打开/关闭集成终端，不用切窗口。

多个终端 `Cmd + Shift + 5` 分屏，左边跑 dev server，右边执行 git 命令，互不干扰。

## Git 实用命令

### stash 临时保存

切分支前有未提交的修改，不想提交又不想丢，用 stash：

```bash
git stash          # 把当前改动暂存
git checkout fix/xxx
# 做完之后
git checkout main
git stash pop      # 恢复之前的改动
```

### 撤销上一次提交（保留改动）

提交了但发现有问题，想重新改再提：

```bash
git reset HEAD~1   # 撤销提交，改动回到工作区
# 修改后重新提交
```

### 查看某行代码谁写的

```bash
git blame src/components/FundCard.tsx
# 每一行前面显示 commit hash、作者、时间
```

找到有问题的行之后，用 commit hash 查当时的改动：

```bash
git show abc1234
```

### cherry-pick 单独应用某个提交

分支 A 有个 bugfix 提交，想搬到分支 B，不用整个 merge：

```bash
git checkout branch-b
git cherry-pick abc1234  # abc1234 是那个 bugfix 的 commit hash
```

## 移动端调试

### iOS Safari 真机调试

用数据线连 Mac，iPhone 上打开 Safari，Mac Safari → 开发菜单 → 选择设备 → 选择页面，能看到真实机器上的 DOM 和 Console，比 Chrome 模拟准确很多。

WeChat 内嵌页调试：微信开发者工具 → 公众号网页，或者用 [vconsole](https://github.com/Tencent/vConsole) 在页面里注入一个调试面板：

```js
// 开发环境注入，生产不打包
if (process.env.NODE_ENV === 'development') {
  import('vconsole').then(({ default: VConsole }) => new VConsole())
}
```

手机上就能看 Console、Network、Storage，解决了 WebView 里没有 DevTools 的问题。

### 快速看真机效果

开发机和手机连同一个 WiFi，起 dev server 时用 `--host 0.0.0.0`：

```bash
npm run dev -- --host 0.0.0.0
# 终端输出：Network: http://192.168.1.x:3000
```

手机浏览器直接访问那个 IP，实时预览，热更新也能用。

## 本地 Mock

后端接口没好、或者想在本地稳定复现某种数据，都需要 mock。下面几种方式按场景选。

### 最轻量：直接在请求层拦截

在 axios 封装里加一个开关，开发环境直接 return 假数据，不用额外依赖：

```js
// src/api/request.js
const USE_MOCK = import.meta.env.VITE_USE_MOCK === 'true'

export async function getFundList(params) {
  if (USE_MOCK) {
    return {
      code: 0,
      data: [
        { id: 'fund_001', name: '沪深 300', nav: 1.234 },
        { id: 'fund_002', name: '纳指 ETF', nav: 2.567 },
      ],
    }
  }
  return request.get('/api/fund/list', { params })
}
```

`.env.local` 里加 `VITE_USE_MOCK=true`，不提交 git，只在本地生效。

### 拦截所有请求：axios 拦截器 + mock 映射

接口多的时候，在拦截器里统一处理，避免每个方法都加判断：

```js
// src/mock/data.js
export const mockMap = {
  'GET /api/fund/list': { code: 0, data: [/*...*/] },
  'POST /api/fund/buy': { code: 0, msg: '申购成功' },
}

// src/api/request.js
import axios from 'axios'
import { mockMap } from '@/mock/data'

const isDev = import.meta.env.DEV

const instance = axios.create({ baseURL: '/api' })

instance.interceptors.request.use(config => {
  if (isDev) {
    const key = `${config.method?.toUpperCase()} ${config.url}`
    if (mockMap[key]) {
      // 取消真实请求，直接返回 mock 数据
      config.adapter = () =>
        Promise.resolve({ data: mockMap[key], status: 200, config })
    }
  }
  return config
})
```

新增 mock 数据只改 `mockMap`，不动请求层逻辑。

### 需要模拟延迟 / 错误状态

测试 loading 态、错误页面时，加个 delay 和随机错误：

```js
function mockAdapter(data, { delay = 300, errorRate = 0 } = {}) {
  return () =>
    new Promise((resolve, reject) => {
      setTimeout(() => {
        if (Math.random() < errorRate) {
          reject({ response: { status: 500, data: { code: 500, msg: '服务器异常' } } })
        } else {
          resolve({ data, status: 200 })
        }
      }, delay)
    })
}

// 使用
config.adapter = mockAdapter({ code: 0, data: [] }, { delay: 800, errorRate: 0.3 })
```

30% 概率返回 500，方便测试错误处理逻辑是否正常。

### 推荐工具：MSW（Mock Service Worker）

以上方案都是在应用代码里 mock，会污染业务逻辑。MSW 在 Service Worker 层拦截，应用代码完全不感知：

```bash
npm install msw --save-dev
npx msw init public/  # 生成 mockServiceWorker.js
```

```js
// src/mock/handlers.js
import { http, HttpResponse } from 'msw'

export const handlers = [
  http.get('/api/fund/list', () => {
    return HttpResponse.json({
      code: 0,
      data: [{ id: 'fund_001', name: '沪深 300', nav: 1.234 }],
    })
  }),
  http.post('/api/fund/buy', async ({ request }) => {
    const body = await request.json()
    return HttpResponse.json({ code: 0, msg: `申购 ${body.fundId} 成功` })
  }),
]

// src/main.js
async function enableMocking() {
  if (import.meta.env.DEV) {
    const { worker } = await import('./mock/browser')
    await worker.start({ onUnhandledRequest: 'bypass' })
  }
}

enableMocking().then(() => createApp(App).mount('#app'))
```

好处是 Network 面板里能看到这些请求（标记为 `from ServiceWorker`），和真实请求体验一致，切换到真实接口只需要停掉 worker，不改业务代码。
