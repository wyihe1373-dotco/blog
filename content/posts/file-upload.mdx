文件上传是几乎每个业务系统都会遇到的功能，看起来简单，实际上涉及 HTTP 协议细节、浏览器 API、并发控制等多个层面。这篇文章从基础原理到常见进阶方案做一次系统梳理。

## 基础：File API 与 FormData

用户选择文件后，浏览器会把它封装成 `File` 对象（继承自 `Blob`），包含文件名、类型、大小等元信息。

```javascript
const input = document.querySelector('input[type="file"]')
input.addEventListener('change', (e) => {
  const file = e.target.files[0]
  console.log(file.name)    // 'photo.jpg'
  console.log(file.size)    // 字节数
  console.log(file.type)    // 'image/jpeg'
  console.log(file.lastModified)  // 时间戳
})
```

`FormData` 是把文件打包成 HTTP 请求体的标准方式：

```javascript
const form = new FormData()
form.append('avatar', file)           // 单文件
form.append('name', 'wyh')            // 可混入普通字段
form.append('docs', file1)
form.append('docs', file2)            // 同名多文件

fetch('/api/upload', { method: 'POST', body: form })
// 不要手动设置 Content-Type！浏览器会自动补上 boundary
```

## 请求头解析：multipart/form-data

上传文件时请求头长这样：

```
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW
```

`boundary` 是随机生成的分隔符，请求体用它把每个字段分开：

```
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="avatar"; filename="photo.jpg"
Content-Type: image/jpeg

<二进制文件内容>
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="name"

wyh
------WebKitFormBoundary7MA4YWxkTrZu0gW--
```

这就是为什么不能手动写 `Content-Type: multipart/form-data`——没有 `boundary` 服务端根本无法解析。

服务端（Node.js）用 `multer` 解析：

```javascript
import multer from 'multer'
const upload = multer({ dest: 'uploads/' })

app.post('/api/upload', upload.single('avatar'), (req, res) => {
  console.log(req.file)   // { fieldname, originalname, size, path, mimetype }
  res.json({ url: `/uploads/${req.file.filename}` })
})
```

## 上传进度

`fetch` 原生不支持上传进度，需要用 `XMLHttpRequest`：

```javascript
function uploadWithProgress(file, onProgress) {
  return new Promise((resolve, reject) => {
    const xhr  = new XMLHttpRequest()
    const form = new FormData()
    form.append('file', file)

    xhr.upload.addEventListener('progress', (e) => {
      if (e.lengthComputable) {
        onProgress(Math.round((e.loaded / e.total) * 100))
      }
    })

    xhr.addEventListener('load', () => resolve(JSON.parse(xhr.responseText)))
    xhr.addEventListener('error', reject)

    xhr.open('POST', '/api/upload')
    xhr.send(form)
  })
}

// 用法
await uploadWithProgress(file, (pct) => setProgress(pct))
```

## 文件预览

**图片预览（本地，无需上传）**

```javascript
// 方案一：URL.createObjectURL —— 性能好，用完要释放
const url = URL.createObjectURL(file)
imgEl.src = url
// 组件卸载时释放
URL.revokeObjectURL(url)

// 方案二：FileReader —— 结果是 base64，可直接存 state
const reader = new FileReader()
reader.onload = (e) => setPreview(e.target.result)
reader.readAsDataURL(file)
```

两者对比：`createObjectURL` 更快、内存占用更小；`FileReader` 生成的 base64 可序列化存储，适合需要把预览数据传给后端的场景。

## 文件校验

上传前在前端做基础校验，减少无效请求：

```javascript
const ALLOWED_TYPES = ['image/jpeg', 'image/png', 'image/webp']
const MAX_SIZE = 5 * 1024 * 1024  // 5MB

function validate(file) {
  if (!ALLOWED_TYPES.includes(file.type)) {
    return '只支持 JPG / PNG / WebP 格式'
  }
  if (file.size > MAX_SIZE) {
    return `文件不能超过 5MB，当前 ${(file.size / 1024 / 1024).toFixed(1)}MB`
  }
  return null
}
```

注意：`file.type` 基于文件后缀推断，可被伪造。**服务端必须再做一次校验**（如用 `file-type` 库读取文件 magic number）。

## 大文件分片上传

文件超过几十 MB 时，整体上传容易超时或失败。分片上传把文件切成小块并行发送，再由服务端合并。

```javascript
const CHUNK_SIZE = 2 * 1024 * 1024  // 2MB 每片

async function uploadChunks(file) {
  const total  = Math.ceil(file.size / CHUNK_SIZE)
  const fileId = crypto.randomUUID()  // 同一文件的所有分片共享同一个 ID

  const tasks = Array.from({ length: total }, (_, i) => {
    const start = i * CHUNK_SIZE
    const chunk = file.slice(start, start + CHUNK_SIZE)  // Blob.slice

    const form = new FormData()
    form.append('fileId', fileId)
    form.append('index', String(i))
    form.append('total', String(total))
    form.append('chunk', chunk)

    return fetch('/api/upload/chunk', { method: 'POST', body: form })
  })

  // 并行上传所有分片
  await Promise.all(tasks)

  // 通知服务端合并
  await fetch('/api/upload/merge', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ fileId, filename: file.name, total }),
  })
}
```

并发数过大会占满带宽或触发服务端限流，实际应限制并发：

```javascript
// 每次最多并行 3 个分片
async function uploadWithConcurrency(chunks, limit = 3) {
  const queue = [...chunks]
  const workers = Array.from({ length: limit }, async () => {
    while (queue.length) {
      await queue.shift()()
    }
  })
  await Promise.all(workers)
}
```

## 断点续传

分片上传的自然延伸：上传前先问服务端"这个文件传到哪了"，跳过已上传的分片。

```javascript
async function resumableUpload(file) {
  // 用文件内容的 hash 作为唯一标识（比文件名可靠）
  const hash   = await calcHash(file)   // 见下方
  const total  = Math.ceil(file.size / CHUNK_SIZE)

  // 查询已上传的分片
  const { uploaded } = await fetch(`/api/upload/status?hash=${hash}`).then(r => r.json())
  const uploadedSet  = new Set(uploaded)  // e.g. [0, 1, 3]

  const tasks = Array.from({ length: total }, (_, i) => {
    if (uploadedSet.has(i)) return null   // 已上传，跳过

    const chunk = file.slice(i * CHUNK_SIZE, (i + 1) * CHUNK_SIZE)
    const form  = new FormData()
    form.append('hash', hash)
    form.append('index', String(i))
    form.append('chunk', chunk)
    return () => fetch('/api/upload/chunk', { method: 'POST', body: form })
  }).filter(Boolean)

  await uploadWithConcurrency(tasks, 3)
  await fetch('/api/upload/merge', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ hash, filename: file.name, total }),
  })
}
```

用 Web Worker 计算文件 hash，避免阻塞主线程：

```javascript
// hash-worker.js
self.onmessage = async ({ data: file }) => {
  const buffer = await file.arrayBuffer()
  const hash   = await crypto.subtle.digest('SHA-256', buffer)
  const hex    = [...new Uint8Array(hash)].map(b => b.toString(16).padStart(2, '0')).join('')
  self.postMessage(hex)
}

// 主线程
function calcHash(file) {
  return new Promise((resolve) => {
    const worker = new Worker('/hash-worker.js')
    worker.onmessage = (e) => { resolve(e.data); worker.terminate() }
    worker.postMessage(file)
  })
}
```

## 方案选型速查

| 场景 | 推荐方案 |
|------|----------|
| 头像、小图（< 5MB） | FormData + fetch，前端预览用 `createObjectURL` |
| 需要进度条 | XMLHttpRequest + `upload.onprogress` |
| 文件 > 50MB | 分片上传 + 并发控制 |
| 网络不稳定 / 大文件 | 断点续传（分片 + hash 校验） |
| 批量上传 | `Promise.allSettled` 收集每个文件的结果，不因单个失败中断 |
