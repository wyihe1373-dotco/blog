写了很长时间 `class` 之后，我以为自己懂继承了。直到复盘项目里的 SDK 代码——看到 `TextSocket extends Socket`、`TenantConfig extends Event` 这些继承结构——才意识到自己只是会用语法，从没真正想过底层在做什么。顺着这个疑问把原型链重新梳理了一遍。

## `__proto__`、`prototype`、`constructor` 三者关系

这三个属性放在一起看容易绕，拆开来理解就清楚了：

- **`prototype`**：只有函数才有，是一个对象，用来挂载这个构造函数的共享方法
- **`__proto__`**：每个对象都有，指向它的原型（即创建它的构造函数的 `prototype`）
- **`constructor`**：`prototype` 上的属性，指回构造函数本身

```js
function Person(name) {
  this.name = name
}

Person.prototype.greet = function () {
  return `Hi, I'm ${this.name}`
}

const p = new Person('wyh')

console.log(p.__proto__ === Person.prototype)        // true
console.log(Person.prototype.constructor === Person) // true
console.log(p.constructor === Person)                // true（通过原型链找到的）
```

原型链的查找规则很简单：访问 `p.greet` 时，先找 `p` 自身有没有，没有就沿着 `__proto__` 往上找，一直到 `null` 为止。

## `new` 做了什么

把 `new` 的步骤搞清楚，很多疑问就自然解开了。`new Foo()` 做了 4 步：

1. 创建一个空对象 `{}`
2. 把这个对象的 `__proto__` 指向 `Foo.prototype`
3. 把 `Foo` 里的 `this` 绑定到这个新对象，执行构造函数
4. 如果构造函数返回的是一个对象，就用那个；否则返回第 1 步创建的对象

用代码模拟一遍，帮助理解：

```js
function myNew(Constructor, ...args) {
  const obj = Object.create(Constructor.prototype)
  const result = Constructor.apply(obj, args)
  return result instanceof Object ? result : obj
}

function Dog(name) {
  this.name = name
}
Dog.prototype.bark = function() {
  return `${this.name} says woof`
}

const d = myNew(Dog, 'Rex')
console.log(d.bark())                       // 'Rex says woof'
console.log(d.__proto__ === Dog.prototype)  // true
console.log(d instanceof Dog)              // true
```

## `class` 是语法糖，底层还是原型

ES6 的 `class` 写起来更干净，但底层和上面是同一套机制。`extends` 做了两件事：子类实例的 `__proto__` 指向父类实例，子类 `prototype` 的 `__proto__` 指向父类 `prototype`。

看两个项目里真实的用法——

**继承 Error，区分业务错误类型：**

```ts
// 交易密码验证被用户取消
export class TradeTokenError extends Error {
  readonly isTradeTokenError = true

  constructor(message = 'Trade token cancelled') {
    super(message)  // 调用 Error 构造函数，设置 message 和调用栈
    this.name = 'TradeTokenError'
  }
}

// 提现提交失败
export class WithdrawSubmitError extends Error {
  constructor(message: string) {
    super(message)
    this.name = 'WithdrawSubmitError'
  }
}

// catch 里用 instanceof 分流处理
try {
  await submitWithdraw()
} catch (err) {
  if (err instanceof TradeTokenError) {
    return  // 用户主动取消，不弹错误提示
  }
  if (err instanceof WithdrawSubmitError) {
    toast.error(err.message)
  }
}
```

`instanceof TradeTokenError` 为什么能识别：`err.__proto__ === TradeTokenError.prototype` 为 true，原型链匹配成立。

**基类抽公共逻辑，子类只管自己的差异：**

WebSocket 管理里，连接、断开、重连是通用的，放到基类；不同 socket 的地址和订阅协议不同，各自在子类实现：

```ts
class Socket {
  connect()    { /* 建立连接 */ }
  disconnect() { /* 断开 */ }
  reconnect()  { /* 断线重连 */ }
  send(msg)    { /* 发消息 */ }
}

class TextSocket extends Socket {
  initSocket = async () => {
    this.ctx.url = await getWsTextUrl()
    this.ctx.socket = new WebSocket(this.ctx.url)
  }
  subscribe(topics: string[])   { /* 订阅行情数据 */ }
  unsubscribe(topics: string[]) { /* 取消订阅 */ }
}

class CspSocket extends Socket {
  initSocket = async () => {
    this.ctx.url = await getWsCspUrl()
    this.ctx.socket = new WebSocket(this.ctx.url)
  }
  subscribe(topics: string[])   { /* 订阅 CSP 数据 */ }
  unsubscribe(topics: string[]) { /* 取消订阅 */ }
}

export const textSocket = new TextSocket()
export const cspSocket  = new CspSocket()

// 两个实例调用 connect()，都走 Socket.prototype.connect
// textSocket.__proto__ → TextSocket.prototype
// TextSocket.prototype.__proto__ → Socket.prototype → 找到 connect
textSocket.connect()
cspSocket.connect()
```

公共方法只写一次，子类通过原型链自动继承，这就是 class 继承真正的价值所在。

## 原型链查找过程

```js
const d = new Dog('Rex')

// d.bark() 的查找路径：
// 1. d 自身 → 没有
// 2. Dog.prototype → 有，返回

// d.toString() 的查找路径：
// 1. d 自身 → 没有
// 2. Dog.prototype → 没有
// 3. Object.prototype → 有，返回
// 4. null → 链的终点，到这里还没找到就是 undefined
```

用 `hasOwnProperty` 可以区分属性是自己的还是继承来的：

```js
console.log(d.hasOwnProperty('name'))  // true（构造函数里赋值的）
console.log(d.hasOwnProperty('bark'))  // false（在原型上）
```

## 原型污染：复盘时看到的一个安全问题

复盘代码时看到某个 merge 工具函数用了 `for...in` 遍历，顺着想了一下，这里其实有原型污染风险：

```js
function merge(target, source) {
  for (const key in source) {
    target[key] = source[key]
  }
}

// 如果 source 来自外部输入（比如接口返回）
const payload = JSON.parse('{"__proto__": {"isAdmin": true}}')
merge({}, payload)

const obj = {}
console.log(obj.isAdmin)  // 预期：undefined
                          // 实际：true ← 从来没赋过值，但它有了
console.log({}.isAdmin)   // true ← 新建的空对象也受影响
console.log([].isAdmin)   // true ← 所有对象都受影响
```

`__proto__` 作为 key 被赋值时，实际上修改了 `Object.prototype`，所有对象都受影响。

修复很简单，用 `Object.keys()` 代替 `for...in`，只遍历对象自有的属性：

```js
function safeMerge(target, source) {
  for (const key of Object.keys(source)) {
    target[key] = source[key]
  }
}

safeMerge({}, payload)
console.log({}.isAdmin)  // undefined ← 正常了
```

## 另外几个有用的场景

**`Object.create(null)` 创建无原型对象**

普通 `{}` 继承自 `Object.prototype`，有 `toString`、`constructor` 等方法。用对象当字典缓存时，如果 key 恰好撞上这些名字就会出问题：

```js
const cache = {}
cache['constructor'] = '某个缓存值'
console.log(cache.constructor)
// 预期：'某个缓存值'
// 实际：function Object() {...}（原型上的 constructor 把自己的覆盖了）

// 用 Object.create(null)，没有原型，完全干净
const pureMap = Object.create(null)
pureMap['constructor'] = '某个缓存值'
console.log(pureMap['constructor'])  // '某个缓存值' ✓
console.log(pureMap.toString)        // undefined，没有继承任何方法
```

**`for...in` 会遍历原型链**

```js
function Animal(name) { this.name = name }
Animal.prototype.type = '动物'

const dog = new Animal('旺财')

for (const key in dog) {
  console.log(key)
  // 'name'  ← 自有属性
  // 'type'  ← 原型上的，通常不想要
}

// 只要自有属性，两种写法：
for (const key in dog) {
  if (Object.hasOwn(dog, key)) console.log(key)  // 只输出 'name'
}
Object.keys(dog)  // ['name']
```

**理解 Vue 2 的 `$set`**

Vue 2 的响应式系统建立在 `Object.defineProperty` 之上。它的工作方式是：组件初始化时，Vue 遍历 `data` 里的每个属性，用 `Object.defineProperty` 给每个属性加上 getter/setter：

```js
// Vue 2 响应式的简化实现
function defineReactive(obj, key, val) {
  Object.defineProperty(obj, key, {
    get() {
      // 收集依赖（记录哪些组件用到了这个属性）
      dep.depend()
      return val
    },
    set(newVal) {
      val = newVal
      // 通知依赖更新（触发视图重新渲染）
      dep.notify()
    }
  })
}

// 初始化时对每个属性执行
Object.keys(data).forEach(key => defineReactive(data, key, data[key]))
```

问题在于：**`Object.defineProperty` 只能拦截已存在的属性**。初始化之后新增的属性，Vue 没有机会给它加 setter，所以赋值时不会触发更新：

```js
// data() { return { obj: { a: 1 } } }

this.obj.a = 2         // ✅ 触发更新（初始化时已经拦截了 a）
this.obj.newKey = 'x'  // ❌ 不触发更新（newKey 没有 setter）
delete this.obj.a      // ❌ 不触发更新（删除也拦截不到）
```

`$set` 就是用来绕过这个限制的。它的内部做了两件事：手动调用 `defineReactive` 给新属性加响应式，然后手动触发一次更新通知：

```js
// Vue 2 $set 的简化实现
Vue.prototype.$set = function(obj, key, value) {
  // 给新属性补上响应式拦截
  defineReactive(obj, key, value)
  // 手动通知依赖更新
  obj.__ob__.dep.notify()
}

// 所以必须这样写才能让视图响应
this.$set(this.obj, 'newKey', 'x')  // ✅ 触发更新
```

**Vue 3 为什么不再需要 `$set`**

Vue 3 把响应式换成了 `Proxy`。`Proxy` 是在对象外包一层代理，对整个对象的操作都会经过这层代理，而不是针对某个具体属性：

```js
const raw = { a: 1 }

const proxy = new Proxy(raw, {
  get(target, key) {
    console.log('读取', key)
    return target[key]
  },
  set(target, key, value) {
    console.log('设置', key, '=', value)
    target[key] = value
    return true
  },
  deleteProperty(target, key) {
    console.log('删除', key)
    delete target[key]
    return true
  }
})

proxy.a = 2          // 设置 a = 2（已有属性，能拦截）
proxy.newKey = 'x'   // 设置 newKey = x（新增属性，也能拦截）
delete proxy.a       // 删除 a（删除，也能拦截）
```

`Proxy` 拦截的是"对这个对象做什么操作"，不需要提前知道有哪些属性。新增、删除、修改都走同一个 trap，Vue 3 的响应式系统在 `set` 和 `deleteProperty` 里触发更新就够了：

```js
// Vue 3 中直接操作即可，无需 $set
this.obj.newKey = 'x'  // ✅ 触发更新
delete this.obj.a      // ✅ 触发更新
```

两者的核心差异就在这里：`Object.defineProperty` 是**属性级别**的拦截，必须提前知道属性名；`Proxy` 是**对象级别**的拦截，任何操作都能捕获到。

---

最后，不要直接给内置对象的原型加方法（如 `Array.prototype.myMethod = ...`），虽然方便，但和 polyfill、第三方库容易冲突。要扩展的话，继承新类或写工具函数更安全。
